# 양현재+ (yangnow+)

수학·과학 문제를 출제하고 풀며 포인트를 쌓는 학습 커뮤니티 플랫폼입니다.  
Next.js 14 + SQLite(Prisma) + Docker 기반으로 동작합니다.

## 주요 기능

| 기능 | 설명 |
|------|------|
| **문제 출제 / 풀기** | 마크다운 + KaTeX 수식 지원, 단일·다중 필수 답변·서술형 문제 |

| **랭킹** | 1·2·3등 금·은·동 색상 강조, 내 순위 배너, 티어 가이드 |
| **대회** | 개인전·팀전 대회, 출제자가 문제 출제, 참가자가 실시간 경쟁 |

| **그룹** | 비공개/공개 그룹, 그룹 내 게시판·채팅·문제집 공유 |
| **문제집** | 문제를 모아 세트로 만들어 공유 |
| **피드** | 커뮤니티 게시판, 좋아요·댓글, 관리자 글 고정 |
| **라이벌** | 라이벌 등록 시 상대가 문제 풀면 알림 |
| **풀이** | 정답자만 풀이 탭 열람 가능, 마크다운+이미지 풀이 작성 |
| **알림** | 라이벌 활동, 문제 승인, 관리자 공지 등 실시간 알림, 모바일/PC 드롭다운 최적화 |
| **테마** | 라이트 / 다크 / 개발자(다크+초록 accent) 3가지 테마 |
| **과목 필터** | 수1·수2·수학·물리·화학·지구·**생명**·정보·증명 등 과목별 필터 |

## 사용법

### 회원가입 / 로그인
- 관리자가 허용한 이메일만 가입 가능 (허용 이메일 목록은 어드민 패널에서 관리)
- 이메일 + 비밀번호 로그인 지원

### 문제 출제
1. **문제** 탭 → **문제 출제** 버튼
2. 제목, 내용(마크다운/KaTeX), 과목, 이미지 첨부
3. 답변 유형 선택:
   - **단일 답변** — 정답 1개 (허용 정답 추가 가능)
   - **다중 필수 답변** — (1), (2)... 각 슬롯 모두 맞아야 정답
   - **서술형** — 글·이미지 제출 → 출제자가 직접 승인/반려
4. 요청 포인트 입력 후 제출 → 관리자 검토 후 승인

### 문제 풀기
- 문제 목록에서 원하는 문제 클릭
- 답 입력 후 제출 (서술형은 글+이미지 첨부)
- 정답 시 포인트 획득, 오답 시 재도전 가능 (재시도 불가 설정된 문제 제외)
- **풀이 탭**은 정답자 / 출제자 / 어드민만 열람 가능

### 대회 참가
1. **대회** 탭에서 진행 예정 대회 확인
2. 개인전: 참가 신청 → 대회 시작 시 문제 공개
3. 팀전: 팀 생성 또는 기존 팀 참가 → 인원 충족 시 문제 공개
4. 제한 시간 내 문제 풀어 점수 획득
5. 종료 후 순위표 공개, 1·2·3등 포인트 시상

### 대회 개최 (출제자 권한 필요)
1. 대회 생성 → 제목·설명·규칙·시간·상금 입력
2. **팀 대회** 설정 시 팀 크기(2~5) 선택
3. 문제 추가 (단일·다중·서술형), 드래그로 순서 변경
4. 출제자·검토자 추가 (닉네임 또는 이메일)
5. 관리자 승인 후 대회 시작 버튼 활성화
6. **서술형 검토 탭**: 제출된 서술형 답안을 문제별 필터로 보고 승인/반려

### 공부 타이머
- **타이머** 탭에서 공부 시작/종료
- 탭을 닫거나 이동해도 타이머가 유지됨 (localStorage 기반)
- 종료 시 자동 저장, 오늘 공부 시간·누적 공부 시간·14일 기록 확인

### 그룹
- 공개 그룹은 누구나 가입, 비공개 그룹은 초대로만 가입
- 그룹 내 게시판, 실시간 채팅, 문제집 공유 가능

### 문제집
- 승인된 문제들을 모아 문제집 생성
- 커버 이미지 업로드 가능
- 공개/비공개 설정, 그룹 공유 가능

### 프로필
- **스트릭 차트**: 년도 단위 ← → 탐색, 연속 활동 일수 뱃지
- **포인트 변화 그래프**: 일별 누적 꺾은선 차트, 년도 단위 탐색
- **기여 분야**: 6축 레이더 차트 (문제 풀기·출제·커뮤니티·학문 기여·대회 출제·대회 참가)
- 공개 프로필 URL: `/profile/{userId}`

## 설치 및 실행

### 사전 요구 사항
- Node.js 18+
- Docker & Docker Compose

### Docker (권장)

```bash
docker-compose up -d --build
```

서버는 기본적으로 `http://localhost:3000`에서 실행됩니다.  
최초 실행 시 `ADMIN_EMAIL`로 지정한 계정이 자동으로 관리자로 등록됩니다.

### 환경 변수

| 변수 | 설명 | 기본값 |
|------|------|--------|
| `DATABASE_URL` | SQLite DB 경로 | `file:/app/data/minlluminati.db` |
| `NEXTAUTH_URL` | 서비스 URL | `http://localhost:3000` |
| `NEXTAUTH_SECRET` | 세션 암호화 키 | (필수) |
| `ADMIN_EMAIL` | 초기 관리자 이메일 | `admin@minlluminati.local` |
| `ADMIN_PASSWORD` | 초기 관리자 비밀번호 | `changeme123` |

### 로컬 개발

```bash
# 1. 의존성 설치
npm install

# 2. 환경 변수 설정 (.env 파일 생성)
DATABASE_URL="file:./dev.db"
NEXTAUTH_SECRET="your-secret"
NEXTAUTH_URL="http://localhost:3000"

# 3. DB 초기화
npx prisma db push

# 4. 개발 서버 실행
npm run dev
```

## 패치 노트

### v1.0 — 마스터 티어 · 팀 채팅 · 모바일 UI 개선 (2026-04-25)
- **마스터 티어** — 상위 1·2·3등은 포인트 무관 마스터 배지, 랭킹 페이지 티어 가이드에 표시
- **최초 루비 배지** — 최초로 2000pt 달성한 유저 이름 옆 이탤릭 `first ruby` 표시
- **팀 채팅** — 팀 대회에서 팀원 전용 채팅 추가, 전체 채팅과 서브탭으로 분리, 대회 전·중·후 항상 열림
- **랭킹 1·2·3등 색상** — 금·은·동 배경색으로 행 강조, 숫자 배지 흰색 텍스트
