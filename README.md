## 상점 & 포인트 시스템

### 상점 포인트 (일반 포인트와 별개)

| 획득 방법 | 내용 |
|-----------|------|
| **문제 승인** | 관리자가 내 문제를 승인하면 `승인 포인트`만큼 상점 포인트 지급 |
| **관리자 지급** | 어드민 패널에서 직접 부여/차감 가능 |

> ⚠️ 자신이 출제한 문제를 직접 풀어도 일반 포인트는 획득할 수 없습니다.

### 상점 아이템

| 아이템 | 설명 | 기본 가격 |
|--------|------|-----------|
| **스트릭 보호막** | 하루 활동을 빠뜨려도 스트릭이 끊기지 않게 보호 | 30 상점 포인트 |
| **프로필 배너** | 프로필 상단에 표시되는 이미지 배너 (관리자가 추가) | 관리자 설정 |

## 스트릭 보호막

**구매**: 프로필 설정 → 포인트 상점 → 스트릭 보호막 구매

**작동 방식**:
1. 하루라도 활동(문제 풀기, 출제, 피드 작성)을 빠뜨리면 스트릭이 끊김
2. 보호막을 보유 중일 때 어제 활동이 없으면 → **자동으로 보호막 1개 소모**하고 스트릭 유지
3. 보호막이 사용된 날은 스트릭 차트에서 **파란색 칸**으로 표시됨
4. 보호막이 없으면 자동 소모 없이 스트릭 그냥 끊김

**공휴일 자동 보호**:
- 매년 4/30 ~ 5/3 (연휴)은 보호막 소모 없이 모든 유저의 스트릭 자동 유지
- 차트에서 파란색으로 표시됨

> 보호막은 **하루에 1개만** 자동 소모됩니다. 여러 날 연속으로 쉬면 각 날마다 1개씩 소모되는 것이 아니라, **가장 최근에 빠진 하루**에만 적용됩니다.

## 사용법

### 회원가입 / 로그인
- 관리자가 허용한 이메일만 가입 가능 (허용 이메일 목록은 어드민 패널에서 관리)
- 이메일 + 비밀번호 로그인 지원

### 문제 출제
1. **문제** 탭 → **문제 출제** 버튼
2. 제목, 내용(마크다운/KaTeX), 과목, 이미지 첨부
3. 답변 유형 선택:
   - **단일 답변** — 정답 1개 (허용 정답 추가 가능)
   - **다중 필수 답변** — (1), (2)... 각 슬롯 모두 맞아야 정답
   - **서술형** — 글·이미지 제출 → 출제자가 직접 승인/반려
4. 요청 포인트 입력 후 제출 → 관리자 검토 후 승인
5. 승인되면 요청 포인트만큼 **상점 포인트** 지급

### 문제 풀기
- 문제 목록에서 원하는 문제 클릭
- 답 입력 후 제출 (서술형은 글+이미지 첨부)
- 정답 시 포인트 획득 (단, 내가 출제한 문제는 포인트 없음)
- 오답 시 재도전 가능 (재시도 불가 설정된 문제 제외)
- **풀이 탭**은 정답자 / 출제자 / 어드민만 열람 가능

### 대회 참가
1. **대회** 탭에서 진행 예정 대회 확인
2. 개인전: 참가 신청 → 대회 시작 시 문제 공개
3. 팀전: 팀 생성 또는 기존 팀 참가 → 인원 충족 시 문제 공개
4. 제한 시간 내 문제 풀어 점수 획득
5. 종료 후 순위표 공개, 1·2·3등 포인트 시상

### 대회 개최 (출제자 권한 필요)
1. 대회 생성 → 제목·설명·규칙·시간·상금 입력
2. **팀 대회** 설정 시 팀 크기(2~5) 선택
3. 문제 추가 (단일·다중·서술형), 드래그로 순서 변경
4. 출제자·검토자 추가 (닉네임 또는 이메일)
5. 관리자 승인 후 대회 시작 버튼 활성화
6. **서술형 검토 탭**: 제출된 서술형 답안을 문제별 필터로 보고 승인/반려

### 공부 타이머
- **타이머** 탭에서 공부 시작/종료
- **뽀모도로 모드**: 25분 집중 → 5분 휴식 → 4사이클마다 15분 긴 휴식 자동 전환
- 과목 태그 선택, 오늘 목표 시간 설정, D-Day 설정
- 오늘 세션 기록 확인 및 삭제, 14일 막대 차트, 과목별 공부 비율

### 그룹
- 공개 그룹은 누구나 가입, 비공개 그룹은 초대로만 가입
- 그룹 내 게시판, 실시간 채팅, 문제집 공유 가능

### 문제집
- 승인된 문제들을 모아 문제집 생성
- 커버 이미지 업로드 가능
- 공개/비공개 설정, 그룹 공유 가능
- 문제집 내 문제에서 뒤로가기 시 문제 목록이 아닌 문제집으로 복귀

### 프로필
- **스트릭 차트**: 년도 단위 ← → 탐색, 연속 활동일수 불꽃 뱃지, 보호막 사용일 파란색 표시
- **포인트 변화 그래프**: 일별 누적 꺾은선 차트, 년도 단위 탐색
- **기여 분야**: 6축 레이더 차트 (문제 풀기·출제·커뮤니티·학문 기여·대회 출제·대회 참가)
- **저장한 문제**: 북마크한 문제 목록 (본인만 보임)
- **포인트 상점**: 상점 포인트 잔액 확인, 보호막·배너 구매·장착
- 공개 프로필 URL: `/profile/{userId}`, 장착한 배너 이미지 표시

## 설치 및 실행

### 사전 요구 사항
- Node.js 18+
- Docker & Docker Compose

### Docker (권장)

```bash
docker compose up --build -d
```

서버는 기본적으로 `http://localhost:3000`에서 실행됩니다.  
최초 실행 시 `ADMIN_EMAIL`로 지정한 계정이 자동으로 관리자로 등록됩니다.

### 환경 변수

| 변수 | 설명 | 기본값 |
|------|------|--------|
| `DATABASE_URL` | SQLite DB 경로 | `file:/app/data/minlluminati.db` |
| `NEXTAUTH_URL` | 서비스 URL | `http://localhost:3000` |
| `NEXTAUTH_SECRET` | 세션 암호화 키 | (필수) |
| `ADMIN_EMAIL` | 초기 관리자 이메일 | `admin@minlluminati.local` |
| `ADMIN_PASSWORD` | 초기 관리자 비밀번호 | `changeme123` |

### 로컬 개발

```bash
# 1. 의존성 설치
npm install

# 2. 환경 변수 설정 (.env 파일 생성)
DATABASE_URL="file:./dev.db"
NEXTAUTH_SECRET="your-secret"
NEXTAUTH_URL="http://localhost:3000"

# 3. DB 초기화
npx prisma db push

# 4. 개발 서버 실행
npm run dev
```

### v1.1 — 보드 안정화 (2026-04-27)

#### 🐛 버그 수정
- **요소 부활 / 스냅백 버그** — 삭제한 요소가 되살아나거나 이동한 요소가 원래 위치로 돌아가는 버그 수정
  - 원인 1: SSE 서버가 저장한 사람에게도 `elements` 이벤트를 echo → 서버 단에서 발신자 echo 차단
  - 원인 2: 저장 요청 2개가 동시에 날아갈 때 오래된 저장이 나중에 완료 → 서버 상태 역전 → 저장 직렬화 큐 추가
  - 원인 3: 세션 ID 미로드 상태에서 `myId = undefined` → 자기 SSE를 남의 것으로 오인 → `myId &&` 가드 추가
- **삭제한 요소가 다른 사람 저장 시 부활** — SSE merge 시 `deletedIdsRef` 미검사 → 내가 삭제한 ID는 원격이 되살릴 수 없게 차단
- **이동 후 다른 사람 SSE로 위치 덮어씌워짐** — `dirtyElementIdsRef`로 수정 중인 요소 추적, 저장 완료 전까지 SSE로 덮어쓰기 불가
- **Application error (클라이언트 예외)** — 여러 원인 수정
  - 긴 획 그릴 때 `Math.min(...pts)` 스프레드 스택 오버플로우 → for 루프로 교체
  - SSE 메시지 `JSON.parse` 예외 → try/catch 추가
  - 이미지 업로드 async 완료 후 컴포넌트 언마운트 시 null 역참조 → null 가드 추가
  - `save()` 네트워크 오류가 unhandled rejection으로 전파 → try/catch 추가
